UC-201 Alta

Reservar Clase En Progreso

Usuario reserva una clase disponible en el calendario.

Actor: Usuario autenticado
Precondición: Clase con cupos disponibles
Postcondición: Reserva confirmada, cupo reducido
Flujo principal:
  1. Usuario navega al calendario
  2. Selecciona clase disponible (verde)
  3. Hace clic en "Reservar"
  4. Modal muestra detalles y confirma
  5. Sistema guarda reserva en localStorage
  6. Actualiza cupos disponibles
  7. Muestra notificación de éxito

UC-202 Alta

Cancelar Reserva En Progreso

Usuario cancela una reserva existente.

Actor: Usuario autenticado
Precondición: Reserva activa existente
Postcondición: Reserva cancelada, cupo liberado
Flujo principal:
  1. Usuario va a "Tus próximas reservas"
  2. Selecciona reserva a cancelar
  3. Hace clic en "Cancelar"
  4. Modal de confirmación aparece
  5. Usuario confirma cancelación
  6. Sistema actualiza estado a "cancelled"
  7. Libera cupo en la clase
  8. Muestra notificación de éxito
UC-203 Media

Ver Historial de Reservas Planeado

Usuario consulta su historial de reservas pasadas.

Actor: Usuario autenticado
Precondición: Reservas existentes en el sistema
Postcondición: Historial mostrado con filtros
Flujo principal:
  1. Usuario hace clic en "Historial"
  2. Sistema carga reservas del usuario
  3. Muestra lista con fechas y estados
  4. Usuario puede filtrar por mes/tipo
  5. Puede ver detalles de cada reserva
UC-204 Media

Unirse a Waitlist Planeado

Usuario se une a la lista de espera de una clase llena.

Actor: Usuario autenticado
Precondición: Clase llena (estado rojo)
Postcondición: Usuario agregado a waitlist
Flujo principal:
  1. Usuario ve clase marcada como "Lleno"
  2. Botón cambia a "Unirse a Waitlist"
  3. Hace clic y confirma
  4. Sistema lo agrega a waitlist
  5. Notifica si hay cupo disponible
