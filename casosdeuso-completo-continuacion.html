3. Administración del Sistema

UC-301 Alta

Gestionar Clases En Progreso

Administrador crea, edita o elimina clases del sistema.

Actor: Administrador
Precondición: Acceso a panel admin
Postcondición: Clase creada/editada/eliminada
Flujo principal (Crear):
  1. Admin navega a "Clases" en panel
  2. Hace clic en "Nueva Clase"
  3. Completa formulario (tipo, horario, coach, capacidad)
  4. Guarda en localStorage
  5. Clase aparece en calendario

UC-302 Media

Gestionar Miembros Planeado

Administrador gestiona usuarios del sistema (ver, editar, suspender).

Actor: Administrador
Precondición: Usuarios registrados
Postcondición: Estado de usuario actualizado
Flujo principal (Editar):
  1. Admin navega a "Miembros"
  2. Busca usuario por nombre/email
  3. Hace clic en "Editar"
  4. Modifica datos (plan, estado)
  5. Guarda cambios
UC-303 Media

Ver Reportes Planeado

Administrador consulta reportes de ocupación, ingresos y tendencias.

Actor: Administrador
Precondición: Datos históricos disponibles
Postcondición: Reporte generado y mostrado
Flujo principal:
  1. Admin navega a "Reportes"
  2. Selecciona período (semana/mes)
  3. Selecciona tipo (ocupación/ingresos)
  4. Sistema genera gráficos
  5. Opción de exportar a PDF/CSV
UC-304 Media

Gestionar Waitlists Planeado

Administrador gestiona listas de espera cuando hay cancelaciones.

Actor: Administrador
Precondición: Clase con waitlist y cupo liberado
Postcondición: Usuario promovido de waitlist a reserva
Flujo principal:
  1. Sistema detecta cupo liberado
  2. Notifica al admin
  3. Admin revisa waitlist
  4. Promueve siguiente en lista
  5. Notifica al usuario automáticamente

4. Sistema de Notificaciones

UC-401 Alta

Notificación de Confirmación En Progreso

Usuario recibe notificación cuando su reserva es confirmada.

Actor: Sistema
Precondición: Reserva exitosa
Postcondición: Notificación mostrada al usuario
Flujo principal:
  1. Usuario completa reserva
  2. Sistema guarda en localStorage
  3. Muestra notificación en pantalla
  4. Guarda notificación en historial
  5. Usuario puede ver detalles
UC-402 Media

Recordatorio de Clase Planeado

Usuario recibe recordatorio antes de su clase programada.

Actor: Sistema
Precondición: Clase programada para hoy
Postcondición: Recordatorio mostrado
Flujo principal:
  1. Sistema verifica reservas del día
  2. 2 horas antes, muestra recordatorio
  3. Incluye detalles (hora, coach, sala)
  4. Opción de cancelar desde notificación
UC-403 Media

Notificación de Waitlist Planeado

Usuario es notificado cuando hay cupo disponible desde waitlist.

Actor: Sistema
Precondición: Usuario en waitlist, cupo liberado
