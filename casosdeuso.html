Introducción

Objetivo: Crear un sitio demo 100% funcional que almacene datos en localStorage del navegador en formato JSON.

Fase actual: Análisis y documentación de casos de uso.

Próxima fase: Implementación interactiva con persistencia en localStorage.

Arquitectura futura: Los datos en JSON serán fácilmente adaptables a un backend (REST API, Firebase, etc.).

1. Autenticación y Perfiles

UC-101 Alta

Login de Usuario Completado

Usuario ingresa con email y password para acceder a su dashboard personal.

Actor: Usuario registrado
Precondición: Usuario tiene cuenta creada
Postcondición: Acceso a dashboard personal
Flujo principal:
  1. Usuario hace clic en "Iniciar Sesión"
  2. Ingresa email y password
  3. Sistema valida credenciales en localStorage
  4. Redirige a user-dashboard.html
  5. Muestra bienvenida personalizada
"user": {
  "id": "user_001",
  "email": "usuario@ejemplo.com",
  "name": "Juan Pérez",
  "role": "member",
  "plan": "premium"
}
UC-102 Alta

Login de Administrador Completado

Administrador ingresa con credenciales especiales para acceder al panel de control.

Actor: Administrador
Precondición: Credenciales admin configuradas
Postcondición: Acceso a admin-dashboard.html
Flujo principal:
  1. Admin ingresa email: admin@barrabox.cl
  2. Cualquier password es aceptado (demo)
  3. Sistema detecta email admin
  4. Redirige a admin-dashboard.html
  5. Muestra panel de administración
UC-103 Media

Registro de Nuevo Usuario Planeado

Nuevo usuario crea una cuenta en el sistema.

Actor: Usuario nuevo
Precondición: Email no registrado
Postcondición: Cuenta creada, acceso automático
Flujo principal:
  1. Usuario hace clic en "Registrarse"
  2. Completa formulario (nombre, email, password)
  3. Sistema guarda en localStorage
  4. Redirige a user-dashboard.html
  5. Muestra tutorial de bienvenida
